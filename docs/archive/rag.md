# RAG 模块记录

## 目标

为 EdgeTalk 增加本地知识库问答能力。

## 基本流程

用户问题
↓
检索本地知识库
↓
构造上下文
↓
本地 LLM 基于资料生成回答

## 当前进度
· 创建 data/knowledge/ 本地知识库
· 支持加载 .txt 文档
· 支持建文档切分为文本块

## 简单检索器

当前第一版 Retriever 使用 TF-IDF 和余弦相似度实现。

为了适配中文文本，使用字符 n-gram 方式进行向量化。

当前方案优点是简单、稳定、容易理解；缺点是语义理解能力有限，后续可升级为 Embedding + 向量数据库。

## RAG 接入 LLM

当前系统已经支持 `/rag-chat` 接口。

整体流程如下：

用户问题
↓
SimpleRetriever 检索本地知识库
↓
取 top_k 个相关文本块
↓
过滤低于 min_score 的弱相关结果
↓
拼接为 context
↓
LocalLLM 基于 context 生成回答

现阶段 Prompt 会要求模型优先根据给定的资料进行回答。

## 当前限制
现阶段版本使用 TF-IDF 进行简单检索，优点是轻量，稳定，容易解释，缺点是语义理解能力有限
